# Skill Writer

Этот навык поможет вам создать хорошо структурированные навыки агента для OpenClaw, соответствующие лучшим практикам и требованиям проверки.

## When to use this Skill

- Создаете новый навык агента
- Пишете или обновляете SKILL.md файлы
- Разрабатываете структуру навыков и интерфейс
- Устраняете проблемы с обнаружением навыков
- Преобразуете существующие подсказки или рабочие процессы в навыки

## Instructions

### Step 1: Определите сферу применения навыков

Сначала разберитесь, что должен делать навык:

1. **Задавайте уточняющие вопросы**:
   - Какие конкретные возможности должен предоставлять этот навык?
   - Когда Клоду следует использовать этот навык?
   - Какие инструменты или ресурсы для этого нужны?
   - Это для личного использования или для совместной работы в команде?

2. **Сосредоточьтесь на этом**: Один навык = одна способность
   - Хорошо: "Заполнение форм в формате PDF", "Анализ данных в Excel"
   - Слишком широко: "Обработка документов", "Инструменты обработки данных"

### Шаг 2: Выберите местоположение навыка

Определите, где создать навык:

**Личные навыки** ("~/.openclaw/workspace/skills/"):
- Индивидуальные рабочие процессы и предпочтения
- Экспериментальные навыки
- Инструменты повышения личной продуктивности

**Проектные навыки** ("~/.openclaw/workspace/skills/"):
- Командные рабочие процессы и соглашения
- Опыт работы в конкретных проектах
- Общие утилиты (привязанные к git)

### Шаг 3: Создайте структуру навыков

Создайте каталог и файлы:

```bash
# Personal
mkdir -p ~/.openclaw/workspace/skills/skill-name

# Project
mkdir -p ~/.openclaw/workspace/skills/skill-name
```

Для получения навыков работы с несколькими файлами:
```
skill-name/
├── SKILL.md (required)
├── reference.md (optional)
├── examples.md (optional)
├── scripts/
│   └── helper.py (optional)
└── templates/
    └── template.txt (optional)
```

### Step 4: Напишите SKILL.md frontmatter

Создайте YAML frontmatterс заполненными обязательными полями `name` и `description`:

```yaml
---
name: skill-name
description: Brief description of what this does and when to use it
---
```
**Требования к полю**:

- **name**:
  - Только строчные буквы, цифры, дефисы
  - Не более 64 символов
  - Должно совпадать с названием каталога
  - Подходит: `pdf-processor`, `git-commit-helper`
  - Плохо: `PDF_Processor`, `Git фиксирует!`

- **description**:
  - Не более 1024 символов
  - Укажите, что он делает И когда его использовать
  - Используйте специальные триггерные слова, которые могли бы сказать пользователи
  - Укажите типы файлов, операции и контекст

### Шаг 5: Напишите эффективные описания

Описание имеет решающее значение для того, чтобы Клод мог раскрыть ваши навыки.

**Формула**: `[Что это делает] + [Когда это использовать] + [Ключевые триггеры]`

**Примеры**:

✅ **Хорошо**:
``yaml
description: Извлекайте текст и таблицы из PDF-файлов, заполняйте формы, объединяйте документы. Используется при работе с PDF-файлами или когда пользователь упоминает PDF-файлы, формы или извлечение документов.
```

✅ **Хорошо**:
``yaml
description: Анализ электронных таблиц Excel, создание сводных таблиц и диаграмм. Используется при работе с файлами Excel, электронными таблицами или анализе табличных данных в формате .xlsx.
```

❌ **Слишком расплывчато**:
``yaml
description: Помогает с документами
description: Для анализа данных
```

**Советы** :
- Укажите конкретные расширения файлов (.pdf, .xlsx, .json).
- Укажите общие фразы пользователей ("анализировать", "извлекать", "генерировать")
- Перечислите конкретные операции (не общие глаголы)
- Добавьте контекстные подсказки ("Использовать, когда...", "Для...")

### Step 6: Структурируйте содержание навыка

Use clear Markdown sections:

```markdown
# Skill Name

Краткий обзор того, что делает этот навык.

## Quick start

Приведите простой пример, чтобы сразу приступить к работе.

## Instructions

Пошаговое руководство для бота:
1. Первый шаг с четкими действиями
2. Второй шаг с ожидаемым результатом
3. Разберитесь с крайними случаями

## Examples

Покажите конкретные примеры использования с помощью кода или команд.

## Best practices

- Основные правила, которым следует следовать
- Распространенные ошибки, которых следует избегать
- Когда использовать, а когда не использовать

## Requirements

Перечислите все зависимости или предварительные требования:
```bash
pip install package-name
```

### Step 7: Add supporting files (optional)

Создайте дополнительные файлы для постепенного раскрытия информации:

**reference.md**: Подробные документы по API, расширенные опции
**examples.md**: Расширенные примеры и варианты использования
**scripts/**: Вспомогательные скрипты и утилиты
**templates/**: Шаблоны файлов или шаблонная таблица

Проверьте эти требования:

✅ ** Структура файла**:
- [ ] SKILL.md существует в правильном расположении
- [ ] Имя каталога совпадает с именем frontmatter

✅ **YAML frontmatter**:
- [ ] Открытие `---` в строке 1
- [ ] Закрывающий "---" перед содержимым
- [ ] Допустимый YAML (без табуляции, с правильным отступом)
- [ ] `название` соответствует правилам именования
- [ ] "описание" должно быть конкретным и содержать < 1024 символа

✅ **Качество контента**:
- [ ] Четкие инструкции для OpenClaw
- [ ] Приведены конкретные примеры
- [ ] Обработаны крайние случаи
- [ ] Перечислены зависимости (если таковые имеются)

✅ **Тестирование**:
- [ ] Описание соответствует вопросам пользователя
- [ ] Навык активируется при выполнении соответствующих запросов
- [ ] Инструкции понятны и доступны для выполнения


### Step 9: Debug if needed

If OpenClaw doesn't use the Skill:

1. **Make description more specific**:
   - Add trigger words
   - Include file types
   - Mention common user phrases

2. **Check file location**:
   ```bash
   ls ~/.openclaw/workspace/skills/skill-name/SKILL.md
   ls ~/.openclaw/workspace/skills/skill-name/SKILL.md
   ```

3. **Validate YAML**:
   ```bash
   cat SKILL.md | head -n 10
   ```

4. **Run debug mode**:
   ```bash
   claude --debug
   ```

## Общие шаблоны

### Умение работать только с текстом

``yaml
---
name: code-reader
description: Чтение и анализ кода без внесения изменений. Используется для проверки кода, понимания кодовых баз или документации.
allowed-tools: Read, Grep, Glob
---
```

### Умение работать со скриптами

``yaml
---
name: data-processor
description: Обрабатывает файлы данных CSV и JSON с помощью скриптов на Python. Используется при анализе файлов данных или преобразовании наборов данных.
---

### Multi-file Skill with progressive disclosure

```yaml
---
name: api-designer
description: Design REST APIs following best practices. Use when creating API endpoints, designing routes, or planning API architecture.
---

## Контрольный список для проверки

Перед окончательной отработкой навыка проверьте его правильность:

- [ ] Название пишется строчными буквами, только через дефис, не более 64 символов
- [ ] Описание должно быть конкретным и содержать менее 1024 символов
- [ ] Описание включает "что" и "когда".
- [ ] Используется YAML frontmatter
- [ ] Инструкции пошаговые
- [ ] Примеры конкретные и реалистичные
- [ ] Зависимости задокументированы
- [ ] В путях к файлам используются прямые косые черты
- [ ] Навык активируется при выполнении соответствующих запросов
- [ ] OpenClaw правильно следует инструкциям

## Устранение неполадок

**Навык не активируется**:
- Сделайте описание более конкретным с помощью триггерных слов
- Укажите типы файлов и операции в описании
- Добавьте предложение "Использовать, когда..." с пользовательскими фразами

**Конфликт нескольких навыков**:
- Сделайте описания более четкими
- Используйте разные триггерные слова
- Сузьте сферу применения каждого навыка

**В Skill есть ошибки**:
- Проверьте синтаксис YAML (без табуляции, с правильным отступом)
- Проверьте пути к файлам (используйте косые черты)
- Убедитесь, что у скриптов есть разрешения на выполнение
- Перечислите все зависимости и убедитесь, что они установлены

## Формат вывода

При создании навыка я буду:

1. Задавать уточняющие вопросы о сфере применения и требованиях
2. Предлагать название навыка и местоположение
3. Создавать файл SKILL.md с соответствующим оформлением
4. Включать четкие инструкции и примеры
5. При необходимости добавлять вспомогательные файлы
6. Предоставьте инструкции по тестированию
7. Проверьте соответствие всем требованиям

Результатом будет полный рабочий навык, соответствующий всем рекомендациям и правилам проверки.
