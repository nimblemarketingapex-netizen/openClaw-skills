# Transcribe Meeting

Бот транскрибирует аудио/видео, экономит токены, сохраняет весь текст и генерирует резюме и задачи без повторной транскрипции.

## Когда использовать

Используй этот скилл, когда пользователь:
- отправляет аудио/видео файл 
- хочет получить транскрипт 
- хочет краткое резюме файла
- хочет подробное резюме с разделением спикеров
- хочет список задач на основе файла

## Примеры запросов

- "Транскрибируй этот файл "
- "Сделай краткое резюме файла"
- "Составь полный протокол файла с разделением спикеров"
- "Составь список задач для команды"
- "Напиши текстом, что было в этом видео"
- "Напиши текстом, что было в этом аудио"

## Шаги

1. Проверка файла:
   - убедиться, что аудио/видео файл получен;
   - поддерживаются форматы: `.wav, .mp3, .ogg, .webm, .mp4`.
2. Конвертация:
   - преобразовать файл в WAV 16kHz моно через `ffmpeg`.
3. Транскрипция:
   - разбить аудио на chunk-и для экономии памяти и токенов;
   - транскрибировать каждый chunk с помощью Faster‑Whisper;
   - собрать полный текст транскрипции.
4. Генерация текстов:
   - вызвать подскиллы:
     - `short_summary` — краткое резюме;
     - `full_summary` — полное резюме с разделением спикеров;
     - `tasks` — список задач для сотрудников.
5. Сохранение результатов:
   - транскрипт и тексты подскиллов сохраняются для последующего использования.

## Структура скилла
- **helpers/** — функции для конвертации, транскрипции, генерации текста и сохранения
- **helpers/summary/** — функции для выбора, какой скилл использовать дальше для генерации краткого и полного резюме и списка задач из папки `subskills`
- **helpers/utils/** — dспомогательные функции для работы с текстовыми файлами и результатами скилла (save_text_to_file и другие утилиты)
- **subskills/** — отдельные подскиллы:
  - short_summary — краткое резюме
  - full_summary — полное резюме
  - tasks — задачи для сотрудников
- **main_skill** — основной исполняемый скрипт для OpenClaw, который вызывает `main_skill_core.py`
- **main_skill_core.py** — основной скрипт для обработки файла и вызова подскиллов
- **requirements.txt** — зависимости
- **SKILL.md** в подскиллах — документация для OpenClaw, чтобы видеть каждый подскилл как отдельный навык

## Использование
1. Положите папку `TranscribeMeetingSkill` в `~/.openclaw/skills/`
2. Убедитесь, что установлены зависимости из `requirements.txt`
3. Вызовите скилл через OpenClaw, отправив аудио/видео файл и запросив нужный результат (транскрипт, краткое резюме, полное резюме или задачи для сотрудников).
4. Результаты сохраняются в текстовых файлах и возвращаются пользователю для дальнейшего использования.
5. Результаты сохрани в `transcript.txt`, `short_summary.txt`, `full_summary.txt` и `tasks.txt` для удобного доступа.
