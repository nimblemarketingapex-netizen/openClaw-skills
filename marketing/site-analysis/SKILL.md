# Site Analysis
## When to use
Когда пользователь присылает ссылку на сайт компании, и требуется оценка его структуры, контента и маркетинговой стратегии.

## Steps

1. **Загрузка страницы**
   - Использовать команду `curl -L {{url}}` или библиотеку `requests` для получения HTML-кода.
   - Опционально: использовать `BeautifulSoup` или `lxml` для парсинга структуры страницы.

2. **Анализ содержания**
   - Определить:
     - Основной продукт/услугу компании.
     - Целевая аудитория (ЦА) и её потребности.
     - Основные офферы и уникальные преимущества (USP).
     - Стиль и тон коммуникации.
   - Выявить слабые места:
     - Непонятная структура или навигация.
     - Недостаток ключевой информации.
     - Медленная загрузка страниц или ошибки верстки.
     - Слабый визуальный дизайн или UX-проблемы.

3. **Генерация предложений**
   - Создать конкретные рекомендации по улучшению:
     - Оптимизация контента (заголовки, тексты, призывы к действию).
     - Улучшение юзабилити и навигации.
     - Идеи для новых офферов и акций.
     - Рекомендации по визуальному оформлению и брендингу.
   - Варианты форматирования ответа:
     - **Краткий обзор:** 3–5 основных предложений.
     - **Подробный план:** детализированные шаги по улучшению контента, структуры и офферов.

4. **Формат ответа**
   - Структурированный JSON или Markdown:
     ```json
     {
       "company_summary": "...",
       "audience": "...",
       "offers": ["..."],
       "weak_points": ["..."],
       "recommendations": ["..."]
     }
     ```
   - Или Markdown-структура с заголовками:
     ```
     ## Company Summary
     ...
     ## Target Audience
     ...
     ## Offers
     ...
     ## Weak Points
     ...
     ## Recommendations
     ...
     ```

## Notes
- Можно подключить NLP-модели для автоматического определения тематики и оффера.
- Рекомендуется использовать дополнительные инструменты проверки скорости сайта и SEO-анализ.
